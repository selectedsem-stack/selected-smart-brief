3 שכבות של Claude.

לחיצה אחת על "שלח לאישור" מפעילה שלוש שכבות עצמאיות — כל אחת אופציונלית, כולן רצות אוטומטית.

LAYER 01

ולידציה

בדיקה לוקלית של שדות חובה — לפני שקוראים ל-AI. אם חסר משהו, האופרטור מקבל הודעה ומלא.

"חסרים שדות חובה: על הלקוח · קהל יעד, PPC · תקציב חודשי"
LAYER 02

ליטוש

Claude מקבל את כל הבריף בקריאה אחת ומחזיר JSON של שדות מלוטשים — שפה מקצועית, ללא שינוי במשמעות.

"אנחנו מוכרים דגים" ↓
"שיווק ומכירת דגים טריים וקפואים, מטבלים ויין"
LAYER 03

עקביות

Claude סורק את כל הסקשנים ביחד, מחפש סתירות בין SEO ל-PPC, מסרים שלא תואמים לקהל, וחוסרים קריטיים.

⚠ קהל ב-SEO: "נשים 25+" · ב-PPC: "נשים וגברים". יישור?

עלות: ~$0.02 לבריף עם prompt caching (Sonnet 4.6). $20 קרדיט חד-פעמי = ~1,000 בריפים = חודשי שימוש בפיילוט.

לערוך בעברית, או לבקש מ-Claude.

מור פותח את האימייל ← לוחץ על הכפתור הכתום ← נכנס למסך עריכה ייעודי. שני אופנים, שניהם בעברית טבעית.

למה זה שונה

מה מיוחד בפתרון הזה.

  • עלות תפעול אפסית. Render free + Resend free + Claude pay-as-you-go = $0 חודשי קבוע. רק $0.02 לבריף בפועל.
  • תבניות מודולריות. בריף חדש (כמו "בניית אתר") = קובץ JSON חדש בלבד. אפס שינויי קוד.
  • עובד גם בלי AI. Graceful degradation מובנה — Claude נופל? המערכת ממשיכה ידנית בלי שבירות.
  • HTML במקום PPTX. RTL עברית מושלמת בלי כלי המרה, פלט קטן (250KB), Ctrl+P לפלט PDF צבעוני מלא.
  • טלפונים נשמרים מהיום. כל המשתתפים מאוחסנים — מוכן ל-WhatsApp Business API בשלב הבא.
  • Audit מלא. כל קריאת AI נרשמת עם input/output tokens — עלות מדויקת בכל רגע.
  • בלי התקנה לצוות. אף אחד לא מתקין app — רק סורק QR מהטלפון.
מה הלאה

מפת דרכים ל-Phase 2.

הבסיס מוכן. ההרחבות מכאן הן הוספת ערך — לא תיקון בעיות.

PHASE 2

WhatsApp Integration

פתיחת קבוצה אוטומטית עם משתתפי הפגישה + שליחת PDF דרך WhatsApp Business API. חוסך 2-3 דקות נוספות.

PHASE 3

בריף בניית אתר

תבנית web-brief עם שדות שונים (פונקציונליות, רפרנסים, מבנה). קובץ JSON חדש בלבד.

PHASE 4

Claude Vision

צילום אוטומטי של ה-PDF + ביקורת ויזואלית של Claude לפני שליחה. שכבת בקרה אחרונה.

PHASE 5

Domain רשמי

מעבר ל-brief.selected.co.il + אימייל מ-brief@selected.co.il.

PHASE 6

אינטגרציית CRM

חיבור הבריפים ל-CRM הקיים — כל בריף הופך אוטומטית ללקוח חדש בפלטפורמה.

PHASE 7

ניתוח לקוחות

דוחות מצטברים — קהלי-יעד הכי נפוצים, סוגי בידולים, פיתוח אסטרטגיה צולבת.
