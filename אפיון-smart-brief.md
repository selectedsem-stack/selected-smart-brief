# Smart Brief — מסמך אפיון מוצר

**חברה:** Selected Digital Marketing Agency  
**תאריך:** 06/05/2026  
**גרסה:** 1.0  
**סטטוס:** ממתין לאישור מנהל

---

## רקע ובעיה

כיום, כל פגישת בריף עם לקוח חדש מתנהלת כך:

1. כל איש צוות מגיע עם מסמך PPTX **מודפס**
2. **כותב ביד** הערות בזמן הפגישה
3. אחרי הפגישה — **מקליד ידנית** לתוך הקובץ
4. **מעביר את הקובץ** מאחד לשני (SEO → PPC → מנהל)
5. המנהל **מאשר ועורך** ידנית
6. **מייצאים PDF ושולחים** ללקוח בוואטסאפ

**תוצאה:** תהליך שלוקח עד 24 שעות, לא מקצועי (ניירות מוכתבים), תלות בין אנשים, ותקשורת מיותרת.

---

## הפתרון — Smart Brief

מערכת web פנימית שמאפשרת מילוי דיגיטלי של הבריף **בזמן הפגישה**, עם אישור מנהל חכם וייצור PDF אוטומטי.

**יעד:** מסיום הפגישה ועד PDF מוכן — פחות מ-10 דקות.

---

## תהליך המשתמש החדש

### שלב 1 — פתיחת בריף (30 שניות לפני הפגישה)

מישהו בצוות נכנס ל-`brief.selected.co.il`, לוחץ **"בריף חדש"**, מזין שם לקוח ובוחר אילו מחלקות משתתפות:

```
[✓] על הלקוח    ← תמיד חובה
[✓] SEO
[✓] PPC
[ ] בניית אתר   ← יתווסף בשלב הבא
```

המערכת מייצרת QR code המוצג על המסך.

### שלב 2 — מילוי בזמן הפגישה

כל איש צוות סורק את ה-QR עם הטלפון האישי (או טאבלט משותף), בוחר את המחלקה שלו, וממלא את הסקשן שלו **במקביל** לשאר — בלי תלות זה בזה.

כל סקשן מכיל:
- **שדות מובנים** בדיוק לפי המסמך הנוכחי (ביטויי SEO, תקציב PPC, קהל יעד וכו')
- **שדה הערות חופשי** בתחתית לכל מה שלא נכנס לשדות

### שלב 3 — עיבוד AI (אוטומטי, לפני שהמנהל רואה)

לחיצה על **"שלח לאישור"** → Claude מריץ 3 שכבות:

1. **ולידציה** — מדגיש שדות חסרים: _"חסר: תקציב PPC, כתובת אתר לקוח."_
2. **ליטוש ניסוחים** — כל שדה עובר שיפור לשפה מקצועית ועברית תקנית. מישהו כתב בשפה עממית — Claude מנסח מחדש ברמת קופירייטר.
3. **בדיקת עקביות** — מוודא שאין סתירות בין סקשנים (קהל יעד, מסרים, בידולים).

הצוות ממלא פערים ושולח. המנהל מקבל חומר שכבר ברמה גבוהה.

### שלב 4 — מסך אישור המנהל

המנהל מקבל **אימייל** עם לינק ייחודי ורואה את כל הבריף בממשק עריכה מלא. שני מצבי עריכה:

| מצב | תיאור |
|-----|--------|
| **עריכה ידנית** | לוחץ על כל שדה ומשנה ישירות |
| **Claude Prompt** | מקליד _"הוסף לסקשן SEO שהלקוח לא היה בגוגל בכלל"_ → Claude מנסח ומוסיף לשדה הנכון |

לאחר עריכה → **"אשר וצור מסמך"** → המערכת מייצרת **HTML מעוצב** בסגנון Selected (RTL מושלם, לוגו, תמונת הצוות, כרטיסיות ממותגות) → המנהל פותח בדפדפן ולוחץ Ctrl+P לשמירה כ-PDF.

**הערת עיצוב:** הוחלט להשתמש ב-HTML ולא ב-PPTX — RTL אמין לחלוטין, עיצוב מדויק, ללא כלי המרה חיצוניים.

**בדיקת אסתטיקה אוטומטית:** לאחר יצירת ה-HTML, המערכת מצלמת את הדפים ושולחת ל-Claude Vision לוידוא שאין שבירות ויזואליות לפני שהמנהל מוריד.

## ארכיטקטורה טכנית

```
[טלפון/טאבלט — טופס Web]
        ↓
[Python Flask Backend]
        ↓
[SQLite — בריפים ונתוני משתתפים]
        ↓
[Claude API — ולידציה + עריכת מנהל]
        ↓
[python-pptx — מילוי תבנית PPTX]
        ↓
[PDF — הורדה ידנית לשליחה בוואטסאפ]
```

**שרת:** שרת קיים של החברה, תת-דומיין `brief.selected.co.il`  
**שפה:** Python (עקביות עם כלי האוטומציה הקיים)  
**DB:** SQLite  
**AI:** Claude API (Anthropic)

## מערכת תבניות (מודולריות)

המערכת בנויה כך שהוספת סוג בריף חדש דורשת **קובץ תבנית בלבד**, ללא שינוי קוד:

```
templates/
  seo-ppc-brief/
    template.pptx     ← ה-PPTX המעוצב עם placeholders
    fields.json       ← הגדרת שדות לכל מחלקה
  web-brief/          ← שלב ב׳ — בניית אתר
    template.pptx
    fields.json
```

`fields.json` מגדיר לכל שדה: שם פנימי, placeholder ב-PPTX, חובה/רשות, תצוגה בעברית.

## הערות נוספות

- ה-PPTX הנוכחי עובר הכנה חד-פעמית: סימון placeholders (`{{client_name}}`, `{{ppc_budget}}` וכו׳) בכל תיבת טקסט
- מספרי הטלפון של משתתפי הפגישה נשמרים מהיום — לשימוש ב-WhatsApp API בשלב ב׳
- הממשק כולו בעברית, RTL
- עובד על כל טלפון/טאבלט מבלי להתקין אפליקציה
